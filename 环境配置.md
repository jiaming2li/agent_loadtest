环境准备：kind、kubectl、jq、docker

step1:
```
KWOK_REPO=kubernetes-sigs/kwok
KWOK_REL=$(curl -s https://api.github.com/repos/$KWOK_REPO/releases/latest | jq -r .tag_name)
# kwok 控制器(替代 kubelet,假 pod 生命周期)
kubectl apply -f "https://github.com/$KWOK_REPO/releases/download/$KWOK_REL/kwok.yaml"
# stage-fast:让 kwok 节点上的 pod 快速变 Running + PodReady
kubectl apply -f "https://github.com/$KWOK_REPO/releases/download/$KWOK_REL/stage-fast.yaml"
```

kwok.yamlCRD 定义 + Controller Deployment告诉 K8s API Server:"存在一种叫 Stage 的自定义资源",并部署 kwok 控制器去监听、执行这些 Stagestage-fast.yamlStage 类型的具体实例(CR)具体规则:"什么样的 Pod/Node,该怎么变化状态"

```
fake-node.yaml(先建 1 个验证,再 scale):

apiVersion: v1
kind: Node
metadata:
  annotations:
    kwok.x-k8s.io/node: fake          # ← kwok 接管这个节点
    node.alpha.kubernetes.io/ttl: "0"
  labels:
    type: kwok                        # ← 给 nodeSelector 用
  name: kwok-node-0
spec:
  taints:
    - key: kwok.x-k8s.io/node
      value: fake
      effect: NoSchedule              # ← 假节点默认 taint
status:
  allocatable: {cpu: "1000", memory: 1Ti, pods: "1000000"}
  capacity:    {cpu: "1000", memory: 1Ti, pods: "1000000"}
  conditions:
    - {type: Ready, status: "True", reason: KubeletReady}
  nodeInfo: {kubeletVersion: fake}
```
